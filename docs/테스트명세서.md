## 1. 테스트 개요

### 1.1 테스트 목적

영업 일일보고 시스템의 기능 요구사항 충족 여부, 권한 제어, 데이터 유효성 검증을 확인한다.

### 1.2 테스트 범위

| 분류 | 테스트 ID 범위 | 케이스 수 |
|---|---|---|
| 인증 | TC-AUTH-001 ~ 008 | 8 |
| 영업사원 마스터 | TC-SP-001 ~ 010 | 10 |
| 고객 마스터 | TC-CUS-001 ~ 009 | 9 |
| 일일보고 | TC-RPT-001 ~ 014 | 14 |
| 방문기록 | TC-VIS-001 ~ 008 | 8 |
| 과제/상담 | TC-PRB-001 ~ 007 | 7 |
| 익일계획 | TC-PLN-001 ~ 007 | 7 |
| 댓글 | TC-CMT-001 ~ 010 | 10 |
| 권한/보안 | TC-SEC-001 ~ 010 | 10 |
| 유효성 검증 | TC-VAL-001 ~ 010 | 10 |
| **합계** | | **93** |

### 1.3 합격 기준

| 등급 | 기준 |
|---|---|
| Pass | 기대 결과와 실제 결과가 일치 |
| Fail | 기대 결과와 실제 결과가 불일치 |
| Block | 사전 조건 미충족으로 테스트 불가 |
| Skip | 해당 환경에서 테스트 제외 |

**출시 기준:** 전체 케이스 Pass율 100%, Fail 0건

## 2. 테스트 환경 및 데이터

### 2.1 테스트 환경

|---|---|

| 팀장 (상급자) | manager@test.com | 상급자 | 팀원 3명 보유 |
| 영업사원 A | sales-a@test.com | 영업사원 | 팀장 직속 부하 |
| 영업사원 B | sales-b@test.com | 영업사원 | 팀장 직속 부하 |

### 2.3 테스트 데이터 (사전 생성 필요)

| 데이터 | ID | 내용 |
|---|---|---|
| 고객 A | customer_id=1 | (주)테스트산업, 활성 |
| 고객 B | customer_id=2 | 비활성고객(주), is_active=false |
